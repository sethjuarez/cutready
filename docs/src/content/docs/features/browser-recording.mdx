## What It Captures

The recorder is designed to capture actions in a browser session while you walk
through a rough demo. The resulting actions are saved as a session for later
review and script refinement.

## Workflow

The planned product flow is:

1. Open a workspace.
2. Choose a detected Chrome or Edge profile, or use a fresh browser.
3. Open the browser and navigate to your demo starting point.
4. Start recording.
5. Perform the walkthrough.
6. Stop recording to save the session and review action cards.

## Expected Capabilities

- **Profile detection** — automatic discovery of Chrome and Edge profiles
- **Fresh browser mode** — use Chromium without personal extensions or sessions
- **Free-form recording** — capture a demo naturally, review afterwards
- **Live action stream** — captured actions appear in real time
- **Multiple takes** — record again without relaunching the browser
- **Saved sessions** — stopped sessions are written into the project

<Aside type="caution" title="Production video is separate">
  FFmpeg screen/audio capture, deterministic replay automation, self-healing
  selectors, and FCPXML timeline export are planned production features. Browser
  recording today captures the interaction trace, not a finished video file.
</Aside>

<Aside type="tip" title="Want to help?">
  Browser recording is easiest to validate with real demo sites and browser
  profiles. If you find a page interaction that is not captured cleanly, open a
  GitHub issue with the browser, page type, and action you expected to see.
</Aside>
